Thrity Network

A free, lightweight browser and network.

no ads no tracking no accounts open source
Download thrity-network0.05.tar.gz

Install

tar -xzf thrity-network0.05.tar.gz
cd thrity-network
bash setup.sh
python3 browser/30web.py

Changelog

v0.05 latest download
  • New: full developer platform — WebKit's built-in Web Inspector is back on (element/CSS/JS console/network/storage), plus a Thrity DevTools window with resolver debugging, page timing, a security panel, and an extensions panel
  • New: a complete extension system with a small sandboxed DSL (no Python execution, no filesystem/network access beyond what's explicitly permitted) — toolbar buttons, menus, panels, page scripts, and 6 browser events extensions can react to
  • New: permission system (ui, devtools, page_info, page_scripts, network, storage) — network access is restricted to the Thrity network only, storage is private and capped per extension, page scripts only load from the extension's own folder
  • New: 3 example extensions included and auto-installed by setup.sh — Developer Helper, Page Tools, Network Monitor — plus a headless CLI (`extensions/cli.py`) to validate and test extensions without launching the browser
  • Fixed: many .onion sites failing with "Internal SOCKSv5 proxy server error" — bare onion addresses now default to plain HTTP instead of forced HTTPS, self-signed onion certs are trusted the same way .thrity certs are, and failed loads on Tor-anonymized tabs now retry automatically since fresh-circuit failures are usually transient
  • Servers: host server gets a `/thrity-info` debug endpoint (uptime, requests served); registry server gets a `/status` endpoint (version, uptime, site/request counts); both support `--verbose` for full per-request timing logs
  • Resolver: new debug-trace mode (`--debug`) showing exactly which step (cache/hosts.json/which registry) answered a lookup and how long each took, same data DevTools' Resolver panel uses
v0.04 download
  • Faster: registries are now queried in parallel instead of one-by-one, and resolved names are cached briefly, so page loads and clicking around a site are noticeably snappier
  • Faster: host server now gzip-compresses pages/CSS/JS on the wire and lets browsers cache static assets for a session
  • Fixes many sites that "didn't work": local storage and video playback (MediaSource) are back on, still wiped when a tab closes, plus a more standard browser identity so fewer sites wrongly block 30web as unsupported
  • A failed .thrity page load now automatically re-resolves and retries once instead of leaving you stuck if a site recently moved
  • Camera/mic/location access is now asked per-site with a prompt instead of silently blocked outright
  • More secure: registry server can run with --https to stop registration secrets from being sent in plain text
  • More secure: stricter validation on .thrity names everywhere (resolver, host server, registry), file writes to the registry are now atomic and race-safe, and lookups are rate-limited too, not just registrations
v0.03 download
  • Anonymize (Tor) is now per-tab, not browser-wide
  • Every tab fully isolated - separate cookies/login state per tab, not just no shared history
  • .thrity sites can now link to each other with thrity://name links, not just be typed in the address bar
  • New "Directory" button browses every site a registry knows about
  • Host server no longer exposes raw folder listings when a page is missing
  • Registry server now rate-limits registrations to slow down name-squatting scripts
v0.02 download
  • Tabs, no saved history — every tab is wiped on close
  • Optional HTTPS for .thrity sites (self-signed, auto-trusted)
  • "Anonymize" toggle routes traffic through Tor
  • Hardened WebKit settings (mic/camera blocked, no DNS prefetch, no local storage)
  • Fixed widespread TLS handshake errors on clearnet sites
v0.01 download
  • Address bar now shows the .thrity name instead of the real IP:port
v0.00 — first release download
  • 30web browser (WebKit2GTK), .thrity resolver, host server, registry server
